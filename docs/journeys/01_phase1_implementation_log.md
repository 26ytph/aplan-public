# 🚀 Phase 1 開發日誌與實作歷程 (Development Log)

**時間：** 2026年3月
**目標：** 打造能支援「自然語言即時推薦」的「臺北時光機」Web 原型。

在本階段（Phase 1），我們成功從無到有建構了應用程式，並處理了語意解析、資料庫搜尋與前端優化的各項技術挑戰。以下記錄了核心的 AI 互動歷程與重要決策。

## 🧠 核心技術探索與決策

1. **Gemini API 深度整合與防護機制**：
   - 全面擁抱全新 `google-genai` SDK，並導入高效能的 `gemini-3.1-flash-lite-preview` 模型以應付黑客松的高併發請求測試。
   - **痛點解決**：為了解決網路不穩或 API 額度耗盡 (Rate Limits) 的突發狀況，我們實作了**堅不可摧的 Fallback 備援防護機制**。當 API 呼叫失敗時能優雅且無縫地降級，確保會場展演流程 100% 絕不中斷。

2. **語意檢索 (Semantic RAG) 的升級**：
   - 將原有的檢索系統從基礎 SQLite 取代，升級至結合 **ChromaDB 向量開發架構**。
   - 系統因此擁有了解析使用者「極端自然語意意圖」（例如：「大雨滂沱想找個安詳的地方」）的能力。這打破了過去用「下拉選單/標籤」死板配對的限制，而能藉由高維度語意對比完成推薦。

3. **魔法介面 (Agentic UI) 與多媒體展示**：
   - 改善了前端靜態圖片在無網路時的破圖問題，建立了本地端圖片備援庫 (Local Fallbacks)。
   - 成功跨越網域限制，將 YouTube 動態影音流暢嵌入景點卡片中，並加入實用的「靜態圖片/影片」快速切換開關，提供兼具豐沛影音與穩定性的極致體驗。
   - **語音連動**：實作 Web Speech API，並結合後端將使用者語音解析為對應的標籤狀態，不僅自動產出文章，更會「自動幫你在介面上勾選好標籤」，給使用者宛如魔法般的操作體驗。
