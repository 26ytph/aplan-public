# 2026YTP 黑客松 – 臺北時光機  Demo (by Antigravity)

## 壹、賽題選擇 A 示範 (行旅台北 / 行樂台北)

### 1. 賽題方向、細分應用、賽題名稱

| 屬性 | 說明 |
| :---- | :---- |
| **賽題方向** | 行旅台北 |
| **賽題名稱** | 即時情境與語意感知之智慧觀光導覽系統 (Antigravity Demo 示範版) |
| **摘要說明** | 本 Demo 系統鼓勵參賽者探索新一代 AI Agentic Development 的方法和工具，打破傳統靜態選項查詢的限制。「臺北時光機」系統整合了 **Hybrid RAG (ChromaDB + SQLite)**、**LLM 動態意圖解析**、**OpenStreetMap 動態地圖退路機制 (Geographic Fallback)** 以及 **YouBike 2.0 即時微交通架構**。<br><br>這套架構精準示範了如何「根據當下的時空背景（天氣、時間、地點），主動將觀光資源串聯並過濾」。然而，**這只是競賽的起點**。正式競賽中，如何針對不同背景和旅遊偏好的遊客，適度的納入更精準且多元的動態旅遊資訊（例如適度考量**住宿安全**等隱性需求），將是留給參賽團隊發揮的空間。 |

### 2. 施政痛點與現況 (Demo 的邊界與真實挑戰)

「臺北時光機」示範了一部分關於「台北市擁有極度豐富的觀光、店家與市集資源」，如何用 AI Agent 去整合運用這些資訊。但在真實的使用情境中，往往面臨更大的挑戰：

官方開放資料 (Open Data) 雖然豐富，卻難以涵蓋瞬息萬變的都會樣貌。例如：那些深藏於巷弄中的特色文化、期間限定的夜間活動，乃至於旅客出遊前高度依賴的各大社群平台即時評價與討論。

有沒有更具創意且合法合規（不侵犯智慧財產權）的技術手段，能在大數據時代下主動搜尋、發掘並整合這些「非標準化、動態且碎片化」的隱藏資訊，交由 AI Agent消化後，精準推送並觸及潛在使用者？

這是本次 Demo 沒有觸及的部份，留待有興趣的人深入研究與大顯身手。

### 3. 目標客群

*   國際觀光客，使用多語言介面，即時翻譯與導覽。
*   追求深度探索、主題式體驗的自由行旅客與年輕族群。
*   臨時決定行程，需要即時獲得情境（如避雨、找夜市）推薦的市民或遊客。
*   關注動態旅遊資訊（包含大型展演、夜間活動、周邊現況，甚至住宿安全等整合資訊）的消費者。

### 4. 預期效益

*   **技術展示效益**：透過本 Demo 展現新世代 AI Agentic 工具在資料處理、意圖解析與即時地理聯動 (Micro-Mobility) 上的強大潛力，鼓勵參賽者跳脫傳統 CRUD 系統思維，勇敢嘗試最新技術。
*   **競賽預期效益 (留給參賽者的挑戰)**：促進深度探索、延長旅客停留時間、真正帶動經濟與商圈消費轉換率，並強化城市智慧觀光品牌形象。

### 5. 關鍵開放資料 (⚠️ 賽前強烈準備建議)

本賽題不限定也不依賴特定的 data.taipei 資料集，但依據本 Demo 專案的開發經驗，**強烈提醒參賽團隊務必提前準備**：

1. **事前盤點與申請 API**：請在競賽日**之前**，找尋並準備好所有可能的外部資料來源 API（例如各大社群 API、天氣 API、地圖 API、交通動態 API 等）。
2. **避免審核延誤**：許多高價值的商業或政府 API 需要數天的金鑰 (API Key) 人工核發流程。千萬不要等到競賽當天才申請，以免白白浪費寶貴的開發時間等待審核。
3. **熟悉索引與調用方法**：提前研究好如何合法抓取這些資料，並構思好索引（Indexing）與調用（Retrieval）的方法架構，以確保競賽當天能迅速與 AI 整合分析。

### 6. 局處內部/特許資料

無（原則不提供去識別化內部資料，請參賽團隊利用自行合法蒐集、網路探勘或情境模擬的資料進行開發與創新驗證）。

### 7. 期待成果形式

✅ **便民服務網頁 (Web Application) / 行動服務 App**：供市民或遊客直接操作互動設計。
✅ **AI 智能代理人 (AI Agents)**：能自主解析語意、規劃動態行程，幫助旅客整合碎裂資訊的虛擬助手。
