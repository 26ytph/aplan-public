# 專案接手工作流 (Project Handoff)

> 當團隊成員啟動新對話並輸入 `/handoff` 時，AI 必須依序執行以下步驟，完成「零遺漏」的上下文同步。

## 執行步驟

### Step 1: 讀取專案大腦 (Sync Project Brain)
// turbo
* 讀取根目錄的 `PROJECT_CONTEXT.md`，同步專案目標、技術棧、目前階段與團隊規範。
* 特別注意 §4「團隊規範與結構約定」，確保後續行為符合約定。

### Step 2: 繼承開發記憶 (Inherit Journey Logs)
// turbo
* 依序讀取 `docs/journeys/` 中的**所有** `.md` 檔案（按檔名排序）。
* 這些是前序開發者與 AI 協同產出的開發日誌，包含了核心技術決策、踩過的坑與解決方案。

### Step 3: 吸收架構決策 (Absorb ADRs)
// turbo
* 讀取 `docs/adr/` 中的所有 ADR 檔案。
* 這些是不可違背的架構約束。

### Step 4: 掃描近期變更 (Recent Changes)
// turbo
* 執行 `git log --oneline -10` 查看最近 10 筆 commit。
* 執行 `git diff --stat HEAD~3` 查看最近 3 筆 commit 的影響範圍。

### Step 5: 產出接手摘要 (Handoff Report)
* 向使用者輸出一份結構化的**接手摘要報告**，包含：
  1. **專案現況**：目前處於哪個 Phase，核心功能有哪些。
  2. **技術棧速覽**：使用了哪些關鍵技術。
  3. **近期重要決策**：從 journeys 和 ADR 中萃取的 2-3 個關鍵決策點。
  4. **近期活動**：最近幾筆 commit 做了什麼。
  5. **待辦或已知問題**：如果日誌中有提及未完成的事項。
* 最後詢問使用者：「接手同步完成！請問接下來要處理什麼任務？」

## 注意事項
* 此工作流的目標是**快速同步**，不要在讀取過程中進行任何修改。
* 如果發現 `docs/journeys/` 為空，提醒使用者：「目前尚無開發日誌，建議先了解專案後產出第一份日誌。」
